# Security Policy

## Intended Use

Zrecon is designed exclusively for:

- Authorized penetration testing
- Security research on assets you own
- Bug bounty programs within scope
- Educational laboratory environments

**Do not use Zrecon against systems you do not own or have explicit written permission to test.**
Unauthorized use may violate local laws, computer abuse laws, and terms of service agreements.

## Reporting Security Issues

If you discover a security vulnerability in Zrecon itself, please report it responsibly:

- Open a GitHub issue marked **[SECURITY]** with a general description
- Do not include exploit code in public issues

## Features Intentionally Not Implemented

Zrecon does not implement:

- Exploitation or payload delivery
- Brute force or credential stuffing
- Authentication bypass techniques
- Denial-of-service or rate-exhaustion attacks
- Stealth scanning, packet spoofing, or evasion techniques
- SYN scanning or raw socket operations

These omissions are intentional and permanent for this version.
